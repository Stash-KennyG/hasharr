purpose and getting started
hasharr is designed to sit in front of manual curation work. It hashes completed downloads, checks for perceptual matches in Stash, and helps remove or tag likely duplicates before you spend time organizing content by hand.
Quick start
1) Open Stash Settings and add one or more Stash GraphQL endpoints.
2) On the Dashboard Playground, tune matching behavior ("Stash Endpoints", "maxTimeDelta", "maxDistance") and test files.
3) Open Hash Service Profiles to configure reusable service profiles and Python script download behavior.
4) In SABnzbd, use the downloaded sab_postProcess.py as a post-process script for your target jobs/categories.
5) Validate outcomes via the Dashboard stats ribbon and the Logs page before relying on full automation.
See the repository README for full Docker examples, API details, and SAB integration notes.
